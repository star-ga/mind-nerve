## Provenance

Eight upstream files hashed into the manifest header (four
`clone_manifest_vN.tsv` + four `clone_log_vN.tsv`, all SHA-256). Plus
121 cloned repos and 2 STARGA-local symlinks (`~/.agents/skills`,
`~/.claude/agents`) whose names are listed in `upstream.txt` alongside
each repo's resolved HEAD commit at freeze time. Any divergence
upstream produces a different `freeze_id` on the next freeze.

## Determinism guarantees

- Canonical order: lex-sorted by per-item `sha256` ascending.
- One item per line, LF endings only.
- JSON: `separators=(",", ":")`, `sort_keys=True`, `ensure_ascii=True`.
- UTF-8 encoded.

Replay-equality: rerunning `freeze.py --version v1.0` against the
same upstream state produces a byte-identical `items.jsonl` and an
identical `freeze_id`. Cross-arch identity follows from the algorithm
being pure SHA-256 + lex-sort, no floats.

## How this is consumed downstream

- `mind-train` (task #56) ingests `items.jsonl` as the supervised
  training corpus.
- `mind-nerve` model bundles bind `freeze_id` into their
  `catalog_hash` slot in the v2 evidence envelope, so every inference
  is verifiably bound to a specific catalog.
- Future freezes (`v1.1`, `v2.0`, …) get their own directory under
  `/data/datasets/mind-nerve-catalog/freeze/` and a sibling doc here.

## Signing

```bash
# When the STARGA-ROOT-2026 key is on-host:
openssl dgst -sha256 -hmac "$STARGA_ROOT_KEY" \
  /data/datasets/mind-nerve-catalog/freeze/v1.0/manifest.json \
  > /data/datasets/mind-nerve-catalog/freeze/v1.0/manifest.sig
```

After signing, this doc should be edited to flip status from
*draft-unsigned* to *signed* and include the signature value (or a
fingerprint of it).

## Open caveats

- v1.0 is the *training* catalog. The schema is fit-for-purpose for
  the Phase 1 training run, not the public-facing one. Public
  redistribution requires per-source-repo license review (task #62).
- English-only; Russian and other languages arrive at v1.1+ after
  multilingual mining.
- The catalog has not yet been pruned for low-quality entries (some
  awesome-list links are dead). Phase 1 training will surface those
  via gradient signal; v1.1 will fold the deadlist back.
