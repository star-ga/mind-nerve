# catalog v1.1-oss + Phase 1 v1.1-oss — OSS-clean retrain

> The public-redistributable variant of catalog-v1.0. Removes
> STARGA-private content, leaked-system-prompts collections, and
> off-domain image-prompt entries.

### Drop reasons (546 items)

| Category | Count | Why excluded |
|---|---|---|

The STARGA private agent inventory (`STARGA/claude-agents` symlink)
contributed 0 items to v1.1-oss because all 60 were filtered. The
v1.0 catalog retains them for the STARGA-tenant dogfood corpus.

## Tokenizer v1.1-oss

- Trained on the v1.1-oss corpus (11,611,131 bytes).
- BPE byte-level, vocab=16,384, same 8 reserved special-token IDs
  as v1.0.
- Training time: 1.58 s.
- `tokenizer_sha256`: `cc2a5502d0fa683c…`.

Drop-in compatible with v1.0 special-token IDs (`[PAD]=0`,
`[BOS]=1`, …, `[SEP]=7`) so any code that hard-coded the slots still
works.

## What this enables

- A `pip install mind-nerve` that doesn't leak commercial content
  into the model bytes.
- A HuggingFace dataset (`star-ga/mind-nerve-catalog-v1.1-oss`) that
  can be republished without per-license worries beyond the upstream
  121-repo composition.
- A clear public/internal boundary: v1.0 stays the STARGA dogfood
  corpus, v1.1-oss is what ships.

## Reproducibility

```bash
# Filter v1.0 → v1.1-oss
python3 ./catalog-builder/filter_oss.py

# Retrain (BPE + Phase 1, ~2 min on a single RTX 3080)
python3 /tmp/retrain_v11.py     # (or check the canonical scripts in catalog-builder/)

# Precompute the route table for v1.1-oss
PYTHONPATH=python python3 -c "
from mind_nerve.inference import precompute_routes
precompute_routes(catalog_path='/data/datasets/mind-nerve-catalog/freeze/v1.1-oss/items.jsonl')
"
```
