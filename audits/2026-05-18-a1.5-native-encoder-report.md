## 1. Build Status

**Build path:** `tools/build_encoder_cdylib.py` (new, replaces broken `tools/build_native_encoder.sh`)

**Why the shell script failed:** `build_native_encoder.sh` used `--output` and `--features` flags that are Cargo build flags, not mindc CLI flags. The correct mindc invocation is `mindc <file> --emit-shared <path>`. Fixed in the Python build driver.

## 2. Source Bugs Fixed

### Bug 1: `sliding_window.mind` arity mismatch

**File:** `mind/kernels/sliding_window.mind`, line 150  
**Bug:** `merge_window_j` called with 5 arguments but defined with 6  
**Fix:** Added missing initial `j=0` argument:
```
- merge_window_j(win_out, global_out, H, t_start, t);
+ merge_window_j(win_out, global_out, H, t_start, t, 0);
```
This caused `mlir-opt` to reject the combined module with `'func.call' op incorrect number of operands for callee`.

### Gap 1: Missing LUT wrapper functions (C shims added)

Three functions referenced by kernel files are not defined in any `.mind` source:

| Symbol | Referenced by | Expected signature |
|--------|--------------|-------------------|
| `tanh_q16(x: i64) -> i64` | `gelu_q16.mind:60` | single-arg wrapper for `tanh_q16_lookup` |
| `rsqrt_q16(x: i64) -> i64` | `layernorm_q16.mind:127`, `l2_norm_q16.mind:57` | single-arg wrapper for `rsqrt_q16_lookup` |
| `softmax_q16(buf, n_rows, T) -> i64` | `encode.mind:292` | row-wise softmax over `n_rows` rows of length `T` |

The MIND LUT files expose a two-phase init+lookup API (`tanh_q16_init()`, `tanh_q16_lookup(handle, x)`) requiring explicit handle management. The kernel files were written expecting single-argument convenience wrappers that were never implemented.

**A1.5 resolution:** C shims in `mind/runtime/lut_shims.c` implement these using libm (`tanh(3)`, `sqrt(3)`, `exp(3)`) in double precision. Statically linked into the `.so`.

**Production requirement:** These shims are NOT bit-identical to the MIND LUT implementations and MUST NOT ship in production. The correct fix is to add handle-management to the kernel files or to write thin MIND wrapper functions that call `init/lookup/free` per-call (correct but slow) or via a handle threaded from `mn_encoder_init` (correct and fast).

## 3. Latency Measurement

### Hardware
- CPU: Intel i7-5930K (6 cores, used up to 4 for this measurement)
- Platform: Linux x86_64

### Corpus
- 1000 queries from `tests/bit_identity/corpus.json`
- Catalog: 11,922 routes × 384 dimensions
- Tokenizer: HuggingFace AutoTokenizer (WordPiece, stays Python-side)

### BLOCKED component: `mn_encoder_encode`

`mn_encoder_encode` cannot be measured because `route_table.q16.bin` (the pre-quantized Q16.16 BERT weight blob, ~142 MB) does not exist in the runtime directory. The offline quantizer (`quantize_phase1_to_q16.py`) has not shipped (Phase 6.2 dependency noted in `c_abi.mind`).

With a null weight blob (`init(0, 0)`), any call to `mn_encoder_encode` segfaults when the embedding lookup dereferences address 0.

### Measured: `mn_encoder_score` + `mn_encoder_topk` (1000 queries, RTLD_LAZY)

| Metric | native (MIND matmul) | numpy BLAS baseline |
|--------|---------------------|---------------------|
| p50    | 14.44 ms            | 0.38 ms             |
| p95    | 15.18 ms            | 3.31 ms             |
| p99    | 16.93 ms            | 5.96 ms             |

The MIND matmul is ~40× slower than numpy BLAS for the 11,922-row × 384-dim inner product. The pure-MIND `matmul_score` kernel is tail-recursive scalar (one multiply-accumulate per recursion step). No SIMD, no loop vectorization at the MLIR level.

### PyTorch baseline: full encode + score + topk (1000 queries)

| Metric | pytorch (sentence-transformers, CPU) |
|--------|--------------------------------------|
| p50    | 86.34 ms                             |
| p95    | 133.59 ms                            |
| p99    | 137.17 ms                            |

### Budget analysis against 30 ms target

- Score + topk (MIND): ~15 ms p95
- Remaining budget for encode: **15 ms** (to meet 30 ms p95 target)
- BERT encoder on 256-token sequence via pytorch CPU: ~70-80 ms (estimated)

The scoring path alone consumes 50% of the 30 ms budget. For the full pipeline to hit 30 ms p95, the MIND BERT encoder must complete in ≤15 ms AND the scoring must be brought down to sub-5 ms (requiring SIMD acceleration).

## 4. Verdict: PARTIAL

### What is working

- `libmind_nerve_encoder.so` builds from mindc v0.6.0 cdylib pipeline.
- All 6 `mn_encoder_*` symbols exported as `T`.
- `mn_encoder_init`, `mn_encoder_free`, `mn_encoder_version` functional.
- `mn_encoder_score` + `mn_encoder_topk` functional with real catalog.
- `.so` is self-contained (no external MIND runtime dependency).

### What is BLOCKED

1. **`mn_encoder_encode` — null weight blob:** `route_table.q16.bin` does not exist. Requires Phase 6.2 offline quantizer to produce the Q16.16 weight blob. Without it, `mn_encoder_encode` segfaults and end-to-end latency cannot be measured.

2. **Score path too slow (40× vs. BLAS):** `mn_encoder_score` / `matmul_score` kernel is tail-recursive scalar. On 11,922-row catalog it takes 15 ms p95 — half the entire 30 ms budget before encode starts. Needs SIMD lowering (AVX2 matmul intrinsics) or numpy fallback for scoring.

3. **Missing LUT wrappers (source design gap):** `tanh_q16`, `rsqrt_q16`, `softmax_q16` convenience wrappers were never implemented in MIND. The C shims are acceptable for measurement but not for production or bit-identity compliance.

4. **Version string stale:** `mn_encoder_version` returns `mindc/0.4.4` embedded as literals in `c_abi.mind`. Should be updated to reflect v0.6.0.

## 5. Required Follow-on Work

### P0 — Unblocks end-to-end latency measurement

| Item | Owner | Estimated effort |
|------|-------|-----------------|
| Ship `quantize_phase1_to_q16.py` offline quantizer → produce `route_table.q16.bin` | Phase 6.2 | 1–2 days |
| Wire Q16.16 weight blob path to `_NativeEncoderRuntime` init | Python-side | 2 hours |

### P1 — Required for p95 ≤ 30 ms

| Item | Description |
|------|-------------|
| MIND matmul SIMD lowering | Replace tail-recursive scalar matmul with AVX2 inner loop in MLIR or via llvm.intr. Required to bring score from 15 ms → <3 ms. |
| LUT wrapper MIND implementations | Add `tanh_q16(x)`, `rsqrt_q16(x)`, `softmax_q16(buf, n_rows, T)` as MIND functions with lazy-init handle caching or handle-threading from mn_encoder_init. |
| Extend handle record | Add 3 LUT handle slots (tanh, rsqrt, softmax) to `c_abi.mind` handle layout. Allocated in `mn_encoder_init`, freed in `mn_encoder_free`. Thread through encode kernel. |

### P2 — Quality / correctness

| Item | Description |
|------|-------------|
| Bit-identity gate with C shims | C shims use IEEE 754 double, not MIND LUT lookup. Cross-arch bit-identity fails until MIND wrappers replace shims. |
| Version string update | `c_abi.mind` hardcodes `mindc/0.4.4` — update to reflect current compiler version. |
| `build_native_encoder.sh` update | Shell script is broken (wrong CLI flags). Either fix to call the Python driver or update to correct mindc CLI syntax. |
