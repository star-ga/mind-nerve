# mind/luts — Q16.16 LUT Family (Phase A1.1)

Five lookup-table kernels that form the transcendental-function layer for
the native MIND inference path.  Every file in this directory is compiled
under mindc v0.4.4 with `std-surface + cross-module-imports` features.

## LUT inventory

| File | Entries | Domain | Q encoding | Role |
|---|---|---|---|---|
| `recip_q32.mind` | 256 | d ∈ [1, 256] (integer) | Q32.32 | Softmax denominator reciprocal |
| `exp_q16.mind` | 4096 | x ∈ [−16, 0] | Q16.16 | Softmax exp (post max-subtract) |
| `tanh_q16.mind` | 4096 | x ∈ [−8, 8] | Q16.16 | GELU tanh approximation |
| `softmax_q16.mind` | — | composing above | Q16.16 | 5-stage softmax kernel |
| `sqrt_q16.mind` | 2048 each | x ∈ [ULP, 256] | Q16.16 | sqrt + rsqrt for LayerNorm / L2-norm |

## Cross-substrate identity contract

Every LUT lookup is a **raw-index** operation: `table[idx]` with no
interpolation.  This is the load-bearing property that makes results
bit-identical across x86, ARM, and CUDA — no FMA ordering dependency
can enter through a pure integer index + load.

If interpolation is ever added for accuracy, the spec requires the fixed
form `lo_val + (hi_val - lo_val) * frac` (subtraction before multiply,
no fused variant).

## Encoding conventions

- **Q16.16**: `i64` where `1.0` == `65536` (`1 << 16`).  Range ≈ ±32768.
- **Q32.32**: `i64` where `1.0` == `4294967296` (`1 << 32`).  Used only
  for the softmax denominator accumulator and `recip_q32`; the extra
  headroom absorbs the up-to-256 denominator sum without overflow.

## ABI

All public functions follow the i64-only ABI: every parameter and return
value is `i64`.  Q16.16 and Q32.32 values are encodings inside `i64`;
no `f32` or `f64` crosses any function boundary.

Each LUT is allocated at runtime via `_init()` (returns an opaque `i64`
handle = heap base address) and released via `_free(handle)`.  The
caller owns the lifecycle.

## Generation

Tables are generated offline by `tools/gen_luts.py` using Python float64
arithmetic with floor-truncation to match the `mindc` truncation discipline.
Do not edit the `__mind_store_i64` call sequences by hand; regenerate:

```
python3 tools/gen_luts.py
```

Validation is built into the generator: every entry is checked against
the float64 reference at generation time; zero ULP errors are the exit
criterion before the files are written.

## Smoke tests

`tests/luts_smoke.mind` checks 32 fixed (input, expected) pairs per LUT
with ≤ 1 ULP-eq tolerance.  The fixture is also generated by `gen_luts.py`
and must be regenerated alongside the LUT files.

## Softmax pipeline (§3.1 of encoder-port-audit)

```
Stage 1: m = max(x_i)                    — sequential, ascending index
Stage 2: y_i = exp_q16_lookup(x_i - m)  — elementwise, LUT
Stage 3: D = sum(y_i) in Q32.32          — sequential, ascending index
Stage 4: r = recip_q32_lookup(D_int)     — LUT
Stage 5: out_i = y_i * r >> 32           — elementwise, Q16.16 narrow
```

`D_int` is `max(1, D >> 32)` — the integer part of the Q32.32 denominator,
clamped to [1, 256] for the recip LUT domain.
